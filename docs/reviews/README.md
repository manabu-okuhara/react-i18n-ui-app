# Review Notes

This folder stores review outputs and audit notes.

Purpose:
- Keep `AGENTS.md` focused on reusable Codex instructions
- Keep review findings in separate, dated documents
- Make it easy to compare audits over time

Suggested naming:
- `i18n-review-YYYY-MM-DD.md`
- `accessibility-review-YYYY-MM-DD.md`
- `performance-review-YYYY-MM-DD.md`

Typical workflow:
1. Ask Codex to review the repo for a topic such as i18n.
2. Save the findings into a dated file in this folder.
3. Track follow-up fixes in GitHub issues or pull requests.
